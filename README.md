# 作業要領書ツール（4ペイン前後）

## フォルダ構成

```
work-manual/
├── README.md                 … この説明
├── 画像/                     … ★ 写真の置き場（iPhone → PC 保存先）
├── reference-photos/         … （旧フォルダ・未使用）
│   └── .gitkeep
├── saved/                    … 完成した要領書 HTML の保存先
│   └── .gitkeep
└── editor.html               … 編集ツール（要サーバー起動）
    css/
    js/
```

## 写真の使い方（運用）

1. iPhone で撮影 → メール・AirDrop 等で PC に保存
2. 画像を **`work-manual/画像/`** に入れる  
   - 例: `01_面取り前.jpg` `02_接着.jpg`
3. 編集ツールで「参考フォルダから選択」またはドラッグ＆ドロップで貼り付け
4. 保存時、要領書用に画像を **`saved/（要領書名）_files/`** へコピーし、HTML から相対パスで参照

※ 1件1 HTML でも、画像は同じフォルダ内の `_files` に置く方式にします（HTML 単体に base64 を埋め込むとファイルが巨大になるため）。

## 保存の段階

| 段階 | 内容 |
|------|------|
| 第1段階 | ブラウザで編集 → `saved/` に HTML をダウンロード保存（ローカル） |
| 第2段階 | 社内サーバー API で `saved/` を一覧・読込・保存 |

## PDF

編集画面の「PDF出力」から、印刷用レイアウトを PDF として保存（ブラウザの PDF 生成を利用）。

---

## A4 レイアウト（確定）

1枚・**横向き**。左に写真4枚（2×2）、右に工程説明6件（2列×3行）、下段に補足。

```
┌─────────────────────────────────────┐
│ タイトル・品番・版・日付              │
├──────────┬──────────────────────────┤
│ 写真     │ ① 工程        ④ 工程      │
│ ①②       │ ② 工程        ⑤ 工程      │
│ ③④       │ ③ 工程        ⑥ 工程      │
├──────────┴──────────────────────────┤
│ 補足説明・注意事項                    │
└─────────────────────────────────────┘
```

| エリア | 内容 |
|--------|------|
| ヘッダー | タイトル・品番・版・日付（＋作成者は任意） |
| 左 2×2 | 写真①〜④（参考フォルダから選択 / D&D） |
| 右 2×3 | 工程①〜⑥の作業説明（箇条書き可） |
| フッター | 補足説明・注意事項 |

### 写真番号と工程説明の書き方（確定）

- 左の写真4枚には **①〜④** の番号バッジを常に表示する（写真の内容とは独立した固定番号）
- 工程①〜⑥の説明では、必要に応じて **「②番写真参照」** のように文章で参照する
- 写真と工程は **1対1で紐づけない**（6工程のうち複数が同じ写真を参照してもよい）

編集画面のヒント例: `例）②番写真を参照し、○○の位置に合わせる`

---

## 使い方

1. ターミナルで `npm start` を実行
2. ブラウザで **http://localhost:3000/work-manual/editor.html** を開く
3. 写真を `画像/` フォルダに入れるか、右パネルへドラッグ＆ドロップ
4. 左の写真マス①〜④・右の工程①〜⑥を入力
5. **保存** → `saved/（ファイル名）.html` と `saved/（ファイル名）_files/`
6. **PDF出力** → 印刷ダイアログで「PDF に保存」
